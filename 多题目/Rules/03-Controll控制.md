# 编程语言规范文档：控制分类 (Controll)

**分类概述**： 一般是在控制类命令之中，是控制程序的指令，更类似于流程的控制，常见的有条件循环、无限循环等，其中一般会伴随包含很多子积木块（代码块），较为复杂且有逻辑关系和衔接关系。

**积木名称规则**: 积木名称是严格的，虽然为自然语言，但每个积木的名称是**严格固定**的，也即下面的每块积木的名称。

**积木语法规则**: 每块积木的运行语法也是有**严格**要求的，其包含的括号如:(){}、以及=>等都是严格的，其次，分号;也是严格的。

**调用规则**: 当你需要引用积木时，需要考虑该积木的含义和作用，调用时必须严格调用其**代码语法**，这个是**严格的**，代码语法中一些非参数部分是不允许改的。

## 1. 积木名称：永远循环
* 积木ID：contoll_loop_unlimited
* 功能描述：无限循环组件，也即无限循环其框架下的命令，层层嵌套，
* 参数定义：无参数
* 代码语法：
  永远循环(() => {/* 内置的执行无限循环的子代码积木块 */});
* 代码示例: 
  * 无限循环旋转15度，也即每次旋转15度执行无限次：永远循环(() => {旋转角度(15);});
  * 无限循环执行每走10步旋转15度：永远循环(() => {移动步数(10);旋转角度(15);});

## 2. 积木名称：有限次循环
* 积木ID：contoll_loop_limited
* 功能描述：有限循环组件，也即执行给定的次数，逻辑与永远循环类似
* 参数定义：
  * 循环次数(n): 数字类型，可以自定义，必须为整数。
* 代码语法：
  重复循环(n, () => {/* 内置的执行限循环的子代码积木块 */});
* 代码示例: 
  * 5次循环旋转15度，也即一共旋转75度：重复循环(5, () => {旋转角度(15);});
  * 5次循环每次执行每走10步旋转15度：重复循环(5, () => {移动步数(10);旋转角度(15);});

## 3. 积木名称：遍历控制
* 积木ID：contoll_for
* 功能描述：有限循环组件，也即执行给定的次数，逻辑与永远循环类似
* 参数定义：
  * 遍历起点(x): 数字类型，可以自定义，必须为整数。
  * 遍历终点(y): 数字类型，可以自定义，必须为整数。
  * 当前指针值(n): 数字类型，可以自定义，必须为整数。
* 代码语法：
  遍历循环(x, y, n, i => {/* 内置的执行的子代码积木块 */});

## 4. 积木名称：条件终止循环
* 积木ID：contoll_loop_until
* 功能描述：有限循环组件，也即循环截至某个条件成立，类似于until。
* 参数定义：
  * 终止条件(xx): 是子代码块。
* 代码语法：
  重复直到(xx, () => {/* 内置的执行循环的子代码积木块 */});

## 5. 积木名称：退出循环
* 积木ID：contoll_droploop
* 功能描述：单一指令，一般是嵌在循环框架中，用于退出循环。
* 参数定义：无参数。
* 代码语法：
  退出循环();

## 6. 积木名称：条件判断
* 积木ID：contoll_if
* 功能描述：类似于if else分支结构，支持单if，也支持if-else，也支持if elif elif等，其中可以支持多个else if。
* 参数定义：
  * 条件(xx): 是子代码块。
* 代码语法：
  * 如果：if (xx) {/* 成立则执行的代码块 */}
  * 如果+否则：if (xx) {/* 成立则执行的代码块 */} else {不成立则执行的代码块}
  * 如果+再如果+则：if (xx) {成立则执行的代码块} else if (xx) {成立则执行的代码块} else {都不成立则执行的代码块}
* 代码示例：
  * 如果角色离开边缘则移动到鼠标处，否则移动到“编程猫”处：if (判断离开边缘("边缘")) {移到("鼠标");} else {移到("编程猫");}
  * 如果角色离开上边缘则移动到鼠标处，如果离开下边缘则移动10步，否则移动到“编程猫”处：if (判断离开边缘("上边缘")) {移到("鼠标");} else if (判断离开边缘("下边缘")) {移动步数(10);} else {移到("编程猫");}

## 7. 积木名称：等待
* 积木ID：contoll_waiting
* 功能描述：单条命令，让当前的对象原地等待若干时间（默认秒）。
* 参数定义：
  * 等待时间(t): 数字类型，默认单位为秒。
* 代码语法：
  * 等待秒数(t);

## 8. 积木名称：等待直到
* 积木ID：contoll_waituntil
* 功能描述：单条命令，让当前的对象原地等待到某个情况达成，类似wait until。
* 参数定义：
  * 条件(xx): 子代码块，可以嵌入。
* 代码语法：
  * 等待直到(xx);

## 9. 积木名称：控制台提示
* 积木ID：contoll_commend
* 功能描述：单条命令，让编译器控制台显示提示，这个提示内容可以自己定义。
* 参数定义：
  * 提示内容(xx): 字符串类型，可以自己定义。
* 代码语法：
  * 打印("已执行");
* 代码示例:
  * 控制台输出“已经完成任务，性能100分”: 打印("已经完成任务，性能100分");
