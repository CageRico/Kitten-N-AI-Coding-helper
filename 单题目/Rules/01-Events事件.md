# 编程语言规范文档：事件分类 (Events)

**分类概述**： 事件积木是所有程序的“入口（Entry Point）”。在代码层面，它们表现为带有回调函数（Callback）的高阶函数。所有后续的动作逻辑必须编写在回调函数的花括号 { } 内部。

**积木名称规则**: 积木名称是严格的，虽然为自然语言，但每个积木的名称是**严格固定**的，也即下面的每块积木的名称。

**积木语法规则**: 每块积木的运行语法也是有**严格**要求的，其包含的括号如:(){}、以及=>等都是严格的，其次，分号;也是严格的。

**调用规则**: 当你需要引用积木时，需要考虑该积木的含义和作用，调用时必须严格调用其**代码语法**，这个是**严格的**，代码语法中一些非参数部分是不允许改的。

## 1. 积木名称：当开始运行
* 积木ID：event_on_start
* 功能描述：当用户点击“开始”按钮时，触发其内部包含的所有程序逻辑。
* 代码语法：
  当开始运行(() => {/* 此处填入内部积木代码 */ });
* 约束规则: 
  * 该积木为顶层积木，不能嵌套在其他任何积木内部。
  * 一个程序通常只允许存在一个“当开始运行”积木。

## 2. 积木名称：当[事件][目标]
* 积木ID：event_on_trigger
* 功能描述：侦测和判断特定角色或对象发生的交互行为。
* 参数定义：
  * 事件(xx)：字符串类型。可选值包括："点击"、"按下"、"放开"。（**只有这三个选项**）。
  * 目标(yy)：字符串类型。可选值包括："自己"、"屏幕"、或其他具体角色 ID（如 "角色1"）。
* 代码语法：
  当触发事件("{{xx}}", "{{yy}}", () => {/* 此处填入内部积木代码 */});
* 代码示例: 
  * 当点击自己：当触发事件("点击", "自己", () => { ... });
  * 当双击背景：当触发事件("双击", "背景", () => { ... });

## 3. 积木名称：当[方向]滑动屏幕
* 积木ID：event_on_slide
* 功能描述：检测用户在屏幕上的手势滑动方向并触发逻辑。
* 参数定义：
  * 方向(xx)：字符串类型。严格限定为以下四个值之一："向上"、"向下"、"向左"、"向右"。
* 代码语法：
  当滑动屏幕("{{xx}}", () => {/* 此处填入内部积木代码 */});
* 代码示例: 
  * 向上滑动：当滑动屏幕("向上", () => { ... });
  * 向左滑动：当滑动屏幕("向左", () => { ... });

## 4. 积木名称：当[触发][按键]
* 积木ID：event_on_print
* 功能描述：检测用户按键盘某些符号而触发的指令。
* 参数定义：
  * 触发(xx)：字符串类型。严格限定为以下四个值之一："按下"、"放开"。
  * 按键(yy)：字符串类型。可以为键盘上的所有符号。
* 代码语法：
  当键盘触发("xx", "yy", () => {/* 此处填入内部积木代码 */});
* 代码示例: 
  * 当按下键盘A：当键盘触发("按下", "A", () => { ... });

## 5. 积木名称：当[状态]碰到[角色]时[角色状态]
* 积木ID：event_on_encounter
* 功能描述：检测用户持续、一下子等状态接触某个角色，这个角色也可以有存在的条件。
* 参数定义：
  * 触发(xx)：字符串类型。严格限定为以下四个值之一："开始"、"持续"、"结束"。
  * 按键(yy)：字符串类型。为角色名，名字可以由玩家自己定义。
  * 角色状态: 可以是其他的积木块，也即子代码。
* 代码语法：
  当碰撞发生("xx", "yy", 角色属性 => {/* 此处填入内部积木代码 */});
* 代码示例: 
  * 当按下键盘A：当键盘触发("按下", "A", () => { ... });

## 6. 积木名称：当条件满足时触发
* 积木ID：event_on_if
* 功能描述：检测当前是否满足所设定的条件，也即本积木块中嵌入的条件是否满足。
* 参数定义：
  * 条件(xx)：可以是子代码块（积木块）。
* 代码语法：
  当条件满足时触发(xx, () => {/* 此处填入满足或不满足执行的内部积木代码 */});
* 代码示例: 
  * 当碰到角色“编程猫”：当条件满足时触发(判断角色碰撞("自己", "编程猫"), () => {});

## 7. 积木名称：当收到广播指令
* 积木ID：event_on_recieve_broadcast
* 功能描述：检测是否收到广播指令，这个广播与之前的状态判断不一样，广播是一类特殊的指令。
* 参数定义：
  * 广播指令(xx)：字符串。可以是自定义或者任意的字符串。
* 代码语法：
  当收到广播("xx", () => {/* 此处填入内部积木代码 */});
* 代码示例: 
  * 当收到广播“hi”：当收到广播("hi", () => {});

## 8. 积木名称: 发送广播
* 积木ID：event_on_broadcasting
* 功能描述：单句命令，发出指定的广播命令。
* 参数定义：
  * 广播指令(xx)：字符串。可以是自定义或者任意的字符串。
* 代码语法：
  发送广播("xx");
* 代码示例: 
  * 发送广播“Hi”：发送广播("Hi");

## 9. 积木名称: 判断是否收到广播
* 积木ID：event_on_recieving_judge
* 功能描述：单句命令，一般是在收到之后做一个判断，仅仅是判断作用。
* 参数定义：
  * 广播指令(xx)：字符串。可以是自定义或者任意的字符串。
* 代码语法：
  判断是否收到广播("xx");
* 代码示例: 
  * 是否收到广播“Hi”：判断是否收到广播("Hi");

## 10. 积木名称: 终止脚本命令
* 积木ID：event_on_truncate
* 功能描述：单句命令，用于对整个脚本进行结尾处理。
* 参数定义：
  * 脚本对象(xx)：字符串。只能是“全部脚本”、“当前脚本”、“当前角色的其他脚本”、“其他角色的脚本”。
* 代码语法：
  停止脚本("xx");
* 代码示例: 
  * 停止全部的脚本也即整个程序：停止脚本("全部脚本");

## 11. 积木名称: 重启
* 积木ID：event_on_restart
* 功能描述：触发启动命令，是一个单独的命令，一般可以用于跳过某些指令然后启动后继续执行。
* 代码语法：
  重启();
